# Интеграция с Git в Faceplate

Faceplate имеет встроенный инструмент для версионного контроля проектов на базе Git. Это позволяет управлять изменениями (коммиты, ветки, откаты) и синхронизироваться с внешними репозиториями (GitLab) прямо из интерфейса студии.

Модуль состоит из трех разделов:
1.  **Settings** (Настройки)
2.  **Local Diff** (Локальные изменения)
3.  **Remote Diff** (Удаленные изменения).

---

## 1. Настройка репозитория (Settings)

Перед началом работы необходимо настроить связь с удаленным репозиторием. Доступно два режима:

### А. Клонирование (Clone)
Используется, если проект уже существует в GitLab и его нужно скопировать в Faceplate. Это удобно для доработки готовых проектов.

* **Username / Password:** Учетные данные GitLab для аутентификации.
* **Remote URL:** HTTPS-ссылка на репозиторий (копируется из интерфейса GitLab).
* **Branch name:** Название рабочей ветки.
* **Project prefix (Контекстная папка):** Путь к папке проекта внутри репозитория. Позволяет работать с конкретным проектом внутри большого репозитория.

### Б. Инициализация (Init)
Используется для создания нового репозитория или привязки текущего локального проекта к пустому проекту в GitLab.

* Заполняются те же поля аутентификации и URL, что и при клонировании.
* Кнопка **Init** инициализирует репозиторий.
* Кнопка **Сброс** появляется после инициализации и позволяет удалить локальный проект, если это необходимо.

---

## 2. Работа с изменениями

### Локальные изменения (Local Diff)
Этот раздел отвечает за управление изменениями на вашей машине.

**Рабочий процесс:**
1.  **Commit (Зафиксировать):** Сохраняет текущие изменения в локальной истории версий. Необходимо ввести сообщение коммита (например, "first commit").
2.  **Push (Загрузить):** Отправляет зафиксированные изменения (коммиты) на удаленный сервер GitLab.

После выполнения команды `Push`, данные обновляются в веб-интерфейсе GitLab.

### Удаленные изменения (Remote Diff)
Раздел предназначен для синхронизации локального проекта с данными на сервере.

**Доступные команды:**
* **Pull (Выгрузить):** Загружает и применяет изменения из удаленного репозитория в ваш локальный проект.
* **Rollback (Откатить):** Возвращает проект к состоянию последнего коммита. **Внимание:** Все незафиксированные изменения, сделанные после последнего коммита, будут безвозвратно потеряны.
* **Rebase (Слияние веток):** Применяет зафиксированные изменения одной ветки к другой, сохраняя порядок изменений.

---

## Рекомендации
* Используйте **Клонирование** для доработки существующих проектов или внедрения готовых компонентов.
* Регулярно выполняйте **Commit**, чтобы сохранять историю изменений.
* Перед началом работы рекомендуется делать **Pull**, чтобы получить актуальную версию проекта от других разработчиков.

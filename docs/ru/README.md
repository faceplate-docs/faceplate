# Общие сведения

FACEPLATE \- программный комплекс, предоставляющий пользователю единое окружение и набор инструментов для сбора и преобразования данных в информацию, её хранения и организации доступа к ней через различные интерфейсы.  
FACEPLATE оснащён развитым web-интерфейсом, предназначенным как для разработчиков, так и для конечных пользователей.

* Разработчики используют интерфейс Studio, в котором выполняется развитие проекта: настройка соединений, проектирование пользовательских интерфейсов, таймсериес данных, математических моделей и т.д.  
* Конечные пользователи используют интерфейс Runtime, через который они получают доступ к информации в режиме реального времени. Это: информация о текущем актуальном состоянии объекта, аналитика, KPI, советы AI, предупреждения о возможных отклонениях и многое другое.

## Использование

FACEPLATE позволяет свести все имеющиеся и непрерывно поступающие данные об объекте в единое информационное пространство и предоставляет следующие возможности:

* визуализация актуальных и архивных данных;  
* выполнение углублённого анализа накопленных и текущих данных с использованием отчётов, графиков и диаграмм;  
* выявление аномалий и корреляций, а также формирование прогнозов с применением высокотехнологичных инструментов, включая методы машинного обучения;  
* получение контекстной информации об объекте управления и рекомендаций по управлению, в том числе вариантов действий в критических ситуациях, на базе LLM-моделей;  
* моделирование различных сценариев развития ситуаций с использованием математических моделей.

## Функционал

Функционал программного комплекса FACEPLATE является гибким и может меняться на этапе согласования технического задания.   
Функционал по модулям программного комплекса:

* Общий вид – основной модуль проекта, в котором отображается иерархический список всех объектов, экранов и компонентов, входящих в проект. Используется для навигации по структуре проекта и быстрого доступа к его элементам.  
* Прототипы – модуль управления прототипами объектов и их отображением. Позволяет создавать, редактировать и переиспользовать типовые элементы проекта, обеспечивая единый стиль и сокращая время разработки  
* Примитивы – модуль управления базовыми элементами проекта  
* Каталоги – модуль для организации данных в виде набора таблиц. Функционал отображения информации в наиболее простой для пользователя форме, организованной в виде набора таблиц  
* Библиотечные модули – модуль создания и управления библиотеками скриптов. Позволяет централизованно хранить и переиспользовать общие функции и логические блоки  
* Локализации – модуль настройки языковой локализации интерфейса. Позволяет управлять переводами текстов и адаптировать проект для многоязычной среды  
* Права – модуль управления доступом пользователей и правами ролей. Используется для разграничения прав на просмотр, редактирование и администрирование элементов проекта  
* Сервера – модуль мониторинга состояния серверов FACEPLATE. Отображает информацию о доступности, загрузке и текущем состоянии серверных компонентов системы  
* Сообщение – модуль системы сообщений. Система сообщений для регистрации и архивирования событий с возможностью их отображения и управления; настраивание категорий сообщений; отображения, оповещения и архивирования  
* Архив – модуль системы архивирования данных. Встроенная подсистема архивирования обеспечивает потребности хранения и обработки данных для различных уровней сложности проектов и позволяет оценить динамику изменения технологических параметров за длительный период времени  
* Агрегаты – редактор создания пользовательских агрегатов для трендов. Позволяет формировать агрегированные показатели и производные данные на основе архивных значений  
* Настройки безопасности – редактор параметров безопасности и авторизации. Используется для настройки протокола OAuth 2.0 и управления механизмами аутентификации и авторизации  
* Calculation – модуль создания вычислительных блоков, включая формулы, правила и алгоритмы обработки данных. Применяется для реализации расчётов и логики обработки сигналов  
* Git – модуль интеграции с системой контроля версий Git. Позволяет управлять версиями разрабатываемого проекта, выполнять коммиты, сравнение и синхронизацию изменений  
* Graph Engine – редактор построения графов, диаграмм и визуального анализа данных. Используется для представления взаимосвязей, потоков данных и аналитических зависимостей  
* IoT соединения – модуль редактирования и настройки IoT-соединений. Предназначен для подключения внешних IoT-устройств и источников данных  
* Драйверы ПЛК – модуль настройки и управления соединениями с программируемыми логическими контроллерами (ПЛК). Обеспечивает интеграцию с промышленным оборудованием  
* Powerfactory service – модуль интеграции с сервисами анализа электрических сетей и моделирования.  
* Реплика – модуль редактирования реплик проекта. Позволяет управлять синхронизацией информации в проекте  
* Скрипт – модуль редактирования пользовательских скриптов. Используется для реализации дополнительной логики, автоматизации и обработки данных  
* Хранилища – модуль настройки баз данных. Используется для организации методов хранения информации

# Запуск

Для успешного запуска сервер должен соответствовать следующим требованиям:  
Серверная среда:

* Операционная система: 64-битные дистрибутивы Linux (Ubuntu, CentOS, Debian, Red Hat или совместимые)  
* Архитектура: x86\_64  
* Оперативная память: от 4 ГБ  
* Дисковое пространство: от 10 ГБ свободного места  
* Сеть: TCP/IP-доступ для подключения клиентских рабочих станций

Клиентская среда:

* Любое устройство с веб\-браузером  
* Веб-браузер последней версии (Apple Safari, Google Chrome, Microsoft Edge, Mozilla Firefox)  
* Поддерживаются мобильные версии браузеров

Общие требования:

* Корректно настроенное системное время и сетевые параметры  
* Возможность доступа к серверу по HTTP/HTTPS  
* Аппаратные ресурсы сервера подбираются в зависимости от предполагаемой нагрузки и масштабов эксплуатации системы

Данные требования достаточны для базового запуска и тестовой эксплуатации FACEPLATE; для промышленных и высоконагруженных систем требуется более производительная конфигурация.  
Путь для папки, с которой будет запущено приложение, должен состоять из символов латинского алфавита, и не должен содержать пробелов, скобок и специальных символов.  
FACEPLATE на сервере запускается из дистрибутива с исполняемым файлом faceplate в директории bin. Для запуска необходимо обладать правами администратора. Чтобы запустить приложение откройте терминал и выполните следующие команды:

* ulimit \-n 200000  
  * необходимо установить лимит на максимальное количество одновременно открытых файлов  
* ./faceplate console  
  * console \- запуск приложения с интерактивной оболочкой

После успешного запуска доступны следующие интерфейсы: FACEPLATE Studio, FACEPLATE Runtime, ECOMET.  
Информация по настройке запускаемой ноды, путей к логам, базе данных и другое доступно по [ссылке](https://github.com/faceplate-docs/faceplate/blob/dev/docs/ru/README.md).

## FACEPLATE Studio

Среда разработки FACEPLATE. Клиент может подключиться с любого устройства если имеет сетевой доступ к серверу FACEPLATE.  
Для этого в браузере необходимо перейти по одному из адресов:

* http://\<ip address\>:9000/fp/studio  
* https://\<ip address\>:9443/fp/studio

## FACEPLATE Runtime

Среда исполнения FACEPLATE, предназначенная для работы операторов. Для открытия среды исполнения в браузере используется один из адресов:

* http://\<ip address\>:9000/fp/runtime  
* https://\<ip address\>:9443/fp/runtime

Где \<ip address\> — IP-адрес сервера, на котором запущен FACEPLATE. Подключение с клиентских устройств осуществляется, если имеется сетевой доступ к серверу FACEPLATE.

# Лицензирование

Лицензия требуется для продолжительной работы сервера. Лицензия выписанная для одного сервера не может быть использована на другом.
